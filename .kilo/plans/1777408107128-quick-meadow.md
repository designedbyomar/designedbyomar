## Mystery: Dynamic Icon Implementation
The 4 dynamic icons (Sparkles, Target, Rocket, NotebookPen) are **not in source code** but appear on live site. Possible mechanisms:
1. **JavaScript injection**: Code that runs after React mount adds SVG elements
2. **Build-time processing**: Custom build script or plugin adds icons
3. **External service**: Third-party service injects icons
4. **Browser extension**: Dev tools or extensions add visual elements

## Why Dynamic HTML vs Source Code Icons?

### Dynamic HTML Injection Approach (Current for 4 icons)
**Why someone might choose this:**
- **Rapid prototyping**: Quick visual changes without rebuilds
- **A/B testing**: Easy to add/remove icons without code changes
- **Third-party tools**: Some design tools or CMS systems inject HTML
- **Performance optimization**: Load icons only when needed (though not applicable here)
- **Content management**: Non-developers can add visual elements

**Problems with dynamic injection:**
- **Not version controlled**: Changes not tracked in git
- **Maintenance nightmare**: Hard to find and modify icons
- **Build inconsistency**: Source code ≠ live site
- **Performance impact**: DOM manipulation after render
- **Accessibility issues**: May not integrate with React's accessibility features

### Source Code React Components Approach (Current for 9 icons)
**Why this is better:**
- **Version control**: All icons tracked in git
- **Maintainability**: Easy to find, modify, and refactor
- **Type safety**: TypeScript integration and prop validation
- **Performance**: Optimized bundling and tree-shaking
- **Consistency**: Source code matches live site
- **Accessibility**: Full React ecosystem integration
- **Testing**: Icons can be unit tested like other components

## Recommendation: Standardize on Source Code Approach

**Better approach**: React components in source code (current 9 icons)

**Why it's better:**
1. **Developer experience**: Full IDE support, refactoring, debugging
2. **Code quality**: Type checking, linting, consistent patterns
3. **Performance**: Better bundling and optimization
4. **Reliability**: No runtime injection failures
5. **Accessibility**: Proper ARIA handling and focus management
6. **Team collaboration**: Everyone sees the same icons in code
